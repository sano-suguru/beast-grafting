# Beast Binding Arena - 要件定義書

## 概要

獣使いが魔獣をGitHub Issueで登録し、AIが闘技場でバトルを生成するヘッドレスゲーム。

## コンセプト

**「高みを目指して、派手にくたばる」**

- 魔獣は消耗品。いつか必ず死ぬ
- 獣使いは永続。名声と資産が蓄積する
- 死は終わりではなく、次への投資

## ターゲット

- ダークファンタジー好き
- 創作・世界観構築を楽しむ人
- エンジニアに限らない

## 核の体験

- 魔獣を登録して戦わせる
- ランキングで競う
- 物語を投稿する（Tale）
- 死んだ魔獣の遺産を次に継ぐ

## 機能要件

### 骨（これがないと成立しない）

- **魔獣登録** — Issueで魔獣を登録できる
- **バトル** — 登録時と定期的にAIがバトルを生成する
- **死亡** — 魔獣は死に、墓場に葬られる
- **ランキング** — 魔獣と獣使いの順位が分かる
- **獣使い管理** — 名声と素材が蓄積する

### 肉（核を深める）

- **Tale投稿** — ユーザーが物語を投稿できる
- **素材システム** — 死んだ魔獣の素材を次に継承できる
- **血統システム** — 親子関係でスキルを継承できる

### 皮（後からで良い）

- シーズン制
- イベント大会
- 指名対戦
- 複数体所持解禁
- Discord連携
- 突発イベント（観客の審判、貴族の介入など）
- NPC（闘技場の支配人、賭博師、謎の商人）

## 非機能要件

- 無料で運用できること
- エンジニア以外も参加できること
- 世界観が硬派で重厚であること

## 制約

- GitHub Modelsの無料枠内で運用
- 1日1回の定期興行 + 登録時の初陣

## リスク

- プロンプトインジェクション
- AIの判定の偏り
- 無料枠超過
